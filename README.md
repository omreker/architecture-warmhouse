# Project_template

## Задание 1. Анализ и планирование

### 1. Описание функциональности монолитного приложения

**Управление отоплением:**

- Пользователь может включить или выключить отопление в своём доме
- Система может отправлять команды на датчики, установленные в домах

**Мониторинг температуры:**

- Пользователь может просматривать текущую температуру в своём доме
- Система может получать данные с датчиков температуры по запросу с сервера
- Данные о температуре записываются и хранятся в базе данных

### 2. Анализ архитектуры монолитного приложения

- Язык программирования: Go
- База данных: PostgerSQL
- Архитектура: Монолит
- Взаимодействие компонентов: Синхронное
- Масштабируемость: Вертикальная (увеличение мощности сервера)
- Развертывание: Полная остановка и перезапуск приложения

### 3. Определение доменов и границы контекстов

**Домен (Управление устройствами):**

- Регистрация устройств и отправка команд

**Домен (Мониторинг устройств):**

- Получение данных с устройств

### **4. Проблемы монолитного решения**

- Трудности в масщтабируемости приложения (может потребоваться много ресурсов)
- Увеличение количества устройств может привести к таймаутам сервера
- Изменения в одной части приложения требует пересборки и тестирования всего монолита
- Ошибка в коде может привести к "падению" всего приложения
